Problem Description

 

You are a software engineer for a factory that produces non-human animated objects such as orcs, goblins, elves, and so on.  Presently, the factory only produces orcs.  As a software engineer, you realize that you need to have a class for orc objects to specify their characteristics and behavior.  Thus, you have Orc.java.  This is your basic class.

 

Further, you realize that you need to have a class for the factory objects.  Yes, right now, you have only one factory.  But your boss is thinking about the future: “Hmm … 10 years from now, I shall have a chain of 50 factories producing these non-human animated objects.”  Thus, for re-usability, you also create a class for the factory objects.  Thus, you have Factory.java. 

 

In addition, your Factory.java class has a “main” method.  This is very important.  It allows user to run the program.  So, a factory foreman can actually run the program.  (Note that Orc.java does not have a “main” method and thus it cannot be run directly.)   Thus, sometimes we call a class like Factory.java an “application” class since it can be run as an application.

 

Further, Factory.java makes use of Orc.java, naturally, since it produces orc objects.  Thus, sometimes we also call a class that uses another class a “tester” class.  In this case, the Factory object will test the Orc objects.

 

Your factory has a user interface allowing the user to specify production modes and corresponding information to specify the production of orcs.

Supplied Solution Components

 

You can download the partially completed programs (Orc.java and Factory.java) from the course web page. To work on the project, open both Orc.java and Factory.java in the Homework1 folder using BlueJ (or IDE of your choice). The Factory.java program has a main method which allows you to run the Factory object. It is imperative to know that some methods and data members may be missing and/or some supplied methods and data members may not be useful. Read the programs carefully.
